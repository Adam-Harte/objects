Objects

Objects are a core data structure in JavaScript that reference a place in memory containing properties (variables) and methods (functions). This allows us to group related data and split code into logical pieces.

Adding, Modifying and Deleting Properties

You can add, modify and delete properties/methods of an object after it has been created. By using the dot notation we can create new properties by defining a new property name after the dot notation e.g. object.newProp. We can also use the dot notation to modify properties by defining the property to modify with the dot notation and initializing it to a new value e.g. object.prop = new value. We can delete properties with the delete key word followed by the property e.g. delete object.prop. Instead of deleting the property from the object you can also modify the value to be null so rather than removing it from the object we are simply resetting it so it exists in case it is needed again later.

Bracket Notation

Objects convert property keys to strings and therefore we can create key names using strings that wouldnt normally convert correctly e.g. one two rather than oneTwo. We can then access these keys via bracket notation [] as dot notation would not allow us access as the spacing would break the declaration.

Shorthand Property Syntax

You can declare object properties with a shorthand syntax when you have variables already defined with the property name you want assigned to your object. Then rather than needing a colon with the value you can just pass the property name and it will automatically pull the value from the variable into that property.

For In Loops

The for in loop lets us loop through the keys of an object similar to for of where we can loop through values.

Spread Operator

The spread operator can be used to spread/copy property values into another object. This can be useful to copy another objects values into a new object witout pointing to the same object reference so that value changes are seperated. Also any propertties you define after spreading that are the same key as anything you spread will overwrite the srpead values, that way you can copy objects and then alter the properties you need.

Object Destructuring

Object destructuring allows us to pull out key value pairs and store them in a variable with the same name as the key. You do this by creating a variable object with all the key values you want to pull out and assign it to that object e.g. { prop1, prop2 } = props. Any remaining propereties you can gorup together in destructuring with the spread operator e.g. { prop1, prop2, ...otherProps } = props. Finally we can rename these desctructured variables by assigning a new name following a colon of the original key name e.g. { prop1: newProp } = props. This allows us to access object properties without the need for the dot ot bracket notations.

Object Assign

The global Object has an assign method which allows us to create or modify objects with copied values from other objects much the same as how we use the sprea operator to do this. The method takes in an argument for the new object {} and then any arguents following are objects whose values you want to copy to this new object.

Checking for property existence

We can use the in state to check if a property exists in an object by asking if the key string is 'in' the object e.g. if ('prop' in object). You can also check if a property is undefined.
